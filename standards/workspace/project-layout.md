# Agent-OS Project Layout

## Rule

Every project under `~/dev/` must have this structure to be considered fully initialized:

```
<project>/
├── AGENTS.md                          # cross-agent policy (source of truth)
├── CLAUDE.md                          # imports AGENTS.md + Claude overlays
├── .claude/commands/agent-os/         # slash commands (from project-install.sh)
└── agent-os/
    ├── product/                       # mission.md, roadmap.md, tech-stack.md
    ├── standards/
    │   └── index.yml                  # required — generated by /index-standards
    └── specs/                         # optional, active work items
```

## Initialization sequence

```bash
cd ~/dev/<project>
~/dev/agent-os/scripts/project-install.sh   # installs commands + standards scaffold
# then in Claude Code:
# /plan-product       → creates agent-os/product/
# /discover-standards → populates agent-os/standards/
# /index-standards    → writes index.yml
```

For existing projects with content: use `project-install.sh --commands-only` to avoid overwriting standards.

## Common mistakes

- **Forgetting `project-install.sh`** — `.claude/commands/agent-os/` is missing, slash commands don't appear
- **Wrong ingest path in claude-os** — passing the project root instead of `agent-os/` results in `total_documents == 0`; always pass `<project>/agent-os` as `profile_path`
